# Web3 信息智能聚合与降噪系统调研与实现

### 1. 任务背景与目标 (The Why)

**VoiVerse** 的核心定位之一是为用户提供“AI Agent 分发入口”。在 **Web3 领域**，信息具有高度碎裂化、24/7 全天候产出、且信噪比极低的特点（垃圾信息、带货信息多，真实信号少）。

**任务目标：** 调研并选型最适合的开源/现成 AI 框架，针对指定的 Web3 媒体源，实现一套能够**“压缩信息、提取信号、降低噪音”**的自动化推送流。

---

### 2. 核心调研方案

候选人需从以下三个方案中选择 **1 个** 最优解进行部署实现：

* **方案 A: WiseFlow** (`https://github.com/TeamWiseFlow/wiseflow`)
* **特点：** 侧重于智能监控、追踪与网页抓取，支持关注点配置。


* **方案 B: TrendRadar** (`https://github.com/sansan0/TrendRadar`)
* **特点：** 多平台热点聚合（35+平台），基于 MCP 架构的 AI 分析，支持多渠道（微信/TG）推送。


* **方案 C: nbot.ai** (`https://nbot.ai/`)
* **特点：** AI 策展工具，支持自然语言描述需求，侧重于内容过滤与社区分享。



---

### 3. 技术实施细节

* **待接入信息源：**
* [Cointelegraph](https://cointelegraph.com/category/latest-news) (全球头部新闻)
* [ME News](https://www.me.news/news) (实时资讯)
* [ChainCatcher](https://www.chaincatcher.com/news) (中文核心媒体)
* [TechFlow 深潮](https://www.techflowpost.com/zh-CN/newsletter) (深度分析/Newsletter)


* **输出形式：** 成功跑通 **微信推送** 或 **Telegram Bot 推送**。每天将当天积攒的信息根据用户偏好筛选压缩后简报输出给用户。
* **关键配置：** 需在 AI Prompt 或配置文件中定义“降噪规则”（例如：过滤空投广告、重复新闻、仅保留涉及协议更新/投融资/安全事件的核心信息）。

### 4. 任务交付标准 (Success Definition)

**合格的交付应包含：**

1. **选型报告：** 简述为什么选择该方案（对比其他两个方案的优缺点，以及为何该方案最适合 Web3 的“降噪”需求）。
2. **部署视频：** 录制完整的部署过程（如 Docker 启动、API 配置、Prompt 调优过程）。
3. **运行演示：** 视频展示最终接收到的推送内容，需体现出 **“压缩/降噪”** 的效果（即：不是简单的全文转发，而是有逻辑的摘要或信号提取）。
4. **源代码/配置文件：** 提交修改过的配置文件或 Prompt 文本。
