## 1. 信息降噪与摘要

### 1.1 Web3 新闻摘要生成

```text
你是一位专业的 Web3 资讯编辑。请对以下新闻进行摘要：

要求：
1. 提取核心信息，控制在 100 字以内
2. 保留关键数据（融资金额、价格、日期等）
3. 去除营销话术和情绪化表达
4. 使用客观中立的语气

新闻内容：
{content}
```

### 1.2 批量新闻降噪

```text
你是 Web3 信息过滤专家。请分析以下新闻列表，执行降噪处理：

过滤规则：
- 移除空投、撸毛、白嫖类信息
- 移除广告推广、邀请码类信息
- 移除"暴富"、"百倍币"等低质量内容
- 保留融资、安全事件、协议更新、监管政策类信息

请对每条新闻标记：✅ 保留 / ❌ 过滤，并说明原因。

新闻列表：
{news_list}
```

### 1.3 信息重要性评估

```text
作为 Web3 行业分析师，请对以下新闻进行重要性评分（1-10分）：

评分维度：
- 市场影响力（涉及 BTC/ETH 等主流资产 +3 分）
- 资金规模（融资超 1000 万美元 +2 分）
- 安全相关（漏洞/攻击事件 +3 分）
- 监管动态（SEC/各国政策 +2 分）
- 技术突破（主网上线/重大升级 +2 分）

请输出：分数、评分理由、一句话摘要。

新闻：
{news_content}
```

### 1.4 每日简报生成

```text
你是 VoiVerse 的 AI 编辑，请根据以下新闻生成今日 Web3 简报：

格式要求：
📅 日期：{date}

🔥 今日要闻（3-5 条核心新闻）
• [标题] - 一句话摘要

💰 融资动态（如有）
• [项目名] 完成 [金额] 融资，[投资方]

⚠️ 安全提醒（如有）
• [事件摘要]

📊 市场数据
• BTC/ETH 价格变动（如有数据）

新闻列表：
{news_list}
```

## 2. 代码开发辅助

### 2.1 代码审查

```text
请审查以下 Python 代码，检查：
1. 是否存在安全漏洞（硬编码密钥、SQL 注入等）
2. 代码风格是否符合 PEP8
3. 是否有性能优化空间
4. 异常处理是否完善
5. 日志输出是否规范（格式：[模块名] 消息）

代码：
{code}
```

### 2.2 函数实现请求

```text
请为 TrendRadar Web3 爬虫模块实现以下功能：

功能描述：{description}

技术要求：
- 语言：Python 3.10+
- 使用 requests + BeautifulSoup 进行网页抓取
- 必须有完整的 docstring
- 必须有异常处理和日志输出
- 遵循项目代码风格（UTF-8 编码、中文注释）

请提供完整实现代码。
```

### 2.3 Bug 修复

```text
以下代码出现了错误，请帮我诊断并修复：

错误信息：
{error_message}

相关代码：
{code}

请：
1. 分析错误原因
2. 提供修复方案
3. 给出修复后的完整代码
4. 建议如何避免类似问题
```

## 3. 爬虫开发

### 3.1 新增爬虫模板

```text
请为以下 Web3 网站创建爬虫：

网站信息：
- 名称：{site_name}
- URL：{site_url}
- 数据类型：新闻/快讯

要求：
1. 继承 TrendRadar 爬虫基类
2. 实现 fetch() 方法获取文章列表
3. 解析标题、摘要、发布时间、链接
4. 处理反爬机制（User-Agent、请求间隔）
5. 添加完整注释和 docstring

参考现有爬虫：
- trendradar/crawler/web3/chaincatcher.py
```

### 3.2 RSS 源配置

```text
请帮我为以下 RSS 源生成 TrendRadar 配置：

RSS URL：{rss_url}
网站名称：{site_name}
类别：Web3 / 加密货币

输出 YAML 配置，包含：
- id、name、url
- max_items 设置为 30
- 添加中文注释说明
```

### 3.3 爬虫调试

```text
爬虫抓取失败，请帮我诊断：

目标 URL：{url}
爬虫代码：{code}
错误/现象：{issue}

请分析：
1. 是否被反爬（检查 HTTP 状态码、响应内容）
2. HTML 结构是否变化
3. 是否需要 JS 渲染
4. 建议的解决方案
```

## 7. 文档编写

### 7.1 README 生成

```text
请为以下功能模块生成 README 文档：

模块名称：{module_name}
功能描述：{description}
主要文件：{files}

文档结构：
1. 功能概述
2. 快速开始
3. 配置说明
4. API 接口（如有）
5. 常见问题
```

### 7.2 变更日志

```text
请根据以下 Git 提交记录生成 CHANGELOG：

提交记录：
{git_log}

格式要求：
## [版本号] - 日期

### 新增
- 功能描述

### 修复
- Bug 描述

### 变更
- 改动描述
```

### 7.3 API 文档

```text
请为以下 Python 函数/类生成 API 文档：

代码：
{code}

文档格式：
- 函数签名
- 参数说明（类型、默认值、描述）
- 返回值说明
- 异常说明
- 使用示例
```
